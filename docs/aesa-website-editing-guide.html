AESA Website Editing Guide

A practical guide for editing, adding images, and making the current AESA React/Vite website more dynamic.

Project: Sarujan2001/AESA
Live domain: https://aesa.site
App folder: my-app

1. How This Website Works

Your site is a React app built with Vite. GitHub Actions builds it and publishes the generated files to GitHub Pages.

File or folderWhat it controls
my-app/src/App.jsxMain website content, pages, navigation, team, calendar, sponsors, contact form behavior, and image URLs.
my-app/src/index.cssAll visual styling: layout, colors, spacing, mobile responsiveness, buttons, cards, and typography.
my-app/src/main.jsxMounts the React app. You usually do not need to edit this.
my-app/public/Files copied directly into the final website, such as CNAME, icons, and public images.
my-app/vite.config.jsBuild settings. For aesa.site, base: '/' is correct.
.github/workflows/deploy.ymlGitHub Pages deployment workflow. You usually do not edit this now that deployment works.
Important: Because the custom domain is aesa.site, keep this in vite.config.js: 
base: '/',
Only use base: '/AESA/' if you go back to https://sarujan2001.github.io/AESA/.

2. The Fast Editing Workflow

  1. Open the project in VS Code.
  2. Edit the content in my-app/src/App.jsx.
  3. Edit the design in my-app/src/index.css.
  4. Test locally with npm.cmd run dev from inside my-app.
  5. Build-test with npm.cmd run build.
  6. Commit and push: 
    git add .
git commit -m "Update website content"
git push origin main
  7. GitHub Actions redeploys the live website automatically.

4. Editing Events, Team, Sponsors, and Past Events

Calendar events

Edit the calendar array. Each row is:

["Date", "Title", "Location", "Time", "Price"]

Example:

const calendar = [
  ["May 7", "Aerospace Industry Night 2026", "Storey Hall, Building 16", "6:30 PM - 9:30 PM", "Free"],
  ["TBA", "Campus Fest / Club Day", "Bowen Street, RMIT City Campus", "Daytime", "Free"],
];

Activities

Edit the activities array. Each row is:

["Activity title", "Small tag", "Description"]

Team members

Edit executiveTeam and generalCommittee. Each row is:

["Name", "Role", "Short description", "Image path or image URL"]

Sponsors

Edit sponsors. Each row is:

["Fallback initials", "Sponsor name", "Sponsor website", "Logo path or logo URL"]

Past events

Edit pastEvents. Each event uses an object:

{
  title: "Aerospace Industry Night 2026",
  year: "2026",
  text: "Short event summary.",
  image: "/images/ain-2026.jpg",
}

5. Adding Your Own Pictures Instead Of Links

You currently use image URLs from services like Unsplash and Clearbit. For your own images, the simplest reliable method is to put image files in my-app/public/images/.

Step-by-step

  1. Create a folder: my-app/public/images.
  2. Put your image there, for example sarujan.jpg.
  3. Use this path in App.jsx: 
    "/images/sarujan.jpg"

Example: Replace a team photo

Before:

["Sarujan Srikaran", "Careers Director", "Public description", "https://images.unsplash.com/photo-example"]

After:

["Sarujan Srikaran", "Careers Director", "Public description", "/images/sarujan.jpg"]

Example: Replace a sponsor logo

Put the logo file here:

my-app/public/images/qantas-logo.png

Then update the sponsor row:

["QF", "Qantas", "https://www.qantas.com/", "/images/qantas-logo.png"]

Recommended image sizes

UseRecommended sizeFormat
Team photos800 x 600 or larger, landscape crop.jpg or .webp
Past event photos1200 x 800 or larger.jpg or .webp
Sponsor logosTransparent logo if possible, around 600 px wide.png, .svg, or .webp
IconsSquare, 512 x 512.png or .svg
Tip: File names should avoid spaces. Use aerospace-night-2026.jpg instead of Aerospace Night 2026.jpg.

6. Making The Site More Dynamic

Right now, the website is static React: content is stored directly inside App.jsx. That is okay for a small club site, but it means every content update requires editing code and pushing to GitHub.

Level 1: Keep it simple, but cleaner

Move the data arrays into a separate file like my-app/src/data/siteData.js. This makes the website easier to edit without scrolling through the component code.

Create:

my-app/src/data/siteData.js

Example:

export const calendar = [
  ["May 7", "Aerospace Industry Night 2026", "Storey Hall, Building 16", "6:30 PM - 9:30 PM", "Free"],
];

Then in App.jsx:

import { calendar } from "./data/siteData.js";

This is the best first improvement for your current website.

Level 2: Use JSON content files

You can store content in JSON files such as:

my-app/src/data/calendar.json
my-app/src/data/team.json
my-app/src/data/sponsors.json

Then import them:

import calendar from "./data/calendar.json";

This is still static, but it separates content from React code and is easier for non-developers to edit carefully.

Level 3: Use a CMS or database

If you want committee members to edit events and photos from a dashboard, use a hosted CMS such as Sanity, Contentful, Decap CMS, or Supabase. The website would fetch content from that service.

This is more powerful, but it adds accounts, API keys, permissions, and extra maintenance.

ApproachBest forComplexity
Current App.jsx arraysSmall site, one editorLow
siteData.jsCleaner code, still simpleLow
JSON filesStructured content editingMedium
CMS/databaseMultiple editors, live dashboardHigh

7. Recommended Refactor For This Website

If you want the site to feel dynamic while keeping GitHub Pages, use siteData.js. GitHub Pages cannot run a backend server, but React can still render dynamic-looking pages from structured data.

Suggested structure:

my-app/src/data/siteData.js
my-app/public/images/team/
my-app/public/images/events/
my-app/public/images/sponsors/

Move these values out of App.jsx:

  • club
  • navItems
  • focusAreas
  • sponsors
  • activities
  • calendar
  • executiveTeam
  • generalCommittee
  • pastEvents
  • benefits

Then import them at the top of App.jsx:

import {
  club,
  navItems,
  focusAreas,
  sponsors,
  activities,
  calendar,
  executiveTeam,
  generalCommittee,
  pastEvents,
  benefits,
} from "./data/siteData.js";

8. Editing The Design

Most styling lives in my-app/src/index.css.

Change brand colors

Edit the CSS variables at the top:

:root {
  --paper: #f5f1e8;
  --ink: #111318;
  --muted: #5f6670;
  --line: #d8d1c4;
  --red: #d71920;
  --charcoal: #171a21;
  --white: #ffffff;
}

For example, changing --red changes many highlight elements across the website.

Change mobile layout

Responsive rules are near the bottom of index.css inside:

@media (max-width: 1040px) { ... }
@media (max-width: 680px) { ... }

Change sponsor logo display

Look for:

.sponsor-logo {
  width: 96px;
  height: 72px;
  object-fit: contain;
}

11. Common Problems

ProblemLikely causeFix
404 on aesa.siteDNS or GitHub Pages custom domain not readyCheck DNS A records, CNAME file, GitHub Pages settings, and wait for propagation.
Images do not showWrong path or file nameUse paths like /images/team/sarujan.jpg and check capitalization.
Build fails on GitHubLocal build would also failRun npm.cmd run build locally and fix the first error.
Site works locally but not liveChanges not pushed or deployment not finishedRun git push origin main and check Actions.
Custom domain disappears after deployNo CNAME in build outputKeep my-app/public/CNAME with aesa.site.

12. Best Next Step

The best improvement for this website is to create my-app/src/data/siteData.js and move all editable content there. That makes future updates easier, keeps the design code separate, and still works perfectly on GitHub Pages.

After that, add your real images into:

my-app/public/images/team/
my-app/public/images/events/
my-app/public/images/sponsors/

Then replace external image URLs in App.jsx or siteData.js with local paths like /images/team/name.jpg.
